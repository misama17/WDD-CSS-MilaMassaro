Ook heb ik voldoende tijd gehad om de sliders meer in dezelfde stijl te maken en beter uit te lijnen. Vlak voordat ik het wilde inleveren, kwam ik erachter dat de responsiveness nog niet helemaal goed werkte, dus ook dit heb ik aangepast. Moest hier even een extra media query aan toevoegen voor de toggles. 
Ik heb ook geoefend met if() in CSS en dit werkte ook, maar mijn CSS bleef wel rood. Daarom heb ik besloten het uiteindelijk op te lossen met een @container style, wat eigenlijk ook wel makkelijker is. 
Dit was mijn code en ik ben wel benieuwd waarom hij in CSS rood bleef:

section:nth-of-type(2) section:nth-of-type(2) div div {
  animation: if(
    style((--speed: 0) and (--rotate: 1)): --rotate-self 2s infinite alternate ease-in-out;

    else: --rotate-self calc(var(--speed) * 1s) infinite alternate ease-in-out;
  );

}

Hier is de nieuwe animatie nog niet in verwerkt, maar het if gedeelte werkte wel. Maar ik liep er wel tegenaan dat ik meerdere dingen moest aanpassen en dan was de @container style veel makkelijker om te gebruiken.


## ALGEMENE REFLECTIE

Ik vind het leuk dat ik iets heb gemaakt dat niemand anders in de klas ook heeft gemaakt. Dit was eigenlijk ook een van de redenen waarom ik deze assignment heb gekozen en niet bijvoorbeelde de rubikscube. Ik denk dat het een samenhangend geheel is en bijvoorbeeld de titel bij de rest past. Bij mijn vorige website die ik inleverde was ik daar niet ze tevreden over, dus ik ben blij dat ik hier nu wel de aandacht aan heb kunnen geven. 
Ik heb in de eerste twee weken vooral heel veel geëxperimenteerd. Eén van mijn leerdoelen is om erachter te komen wat allemaal mogelijk is met bijvoorbeeld CSS. Daarom wilde ik in het begin vooral heel erg kijken naar allemaal varianten van eenzelfde concept. Dit ging een beetje met vallen en opstaan, vooral omdat ik nog nooit echt had gerekend met CSS en ik niet zo goed wist hoe ik variabele en sibling-index() goed in kon zetten. Maar toen ik dat eenmaal een beetje onder de knie had kon ik veel verschillende dingetjes en ideetje uitwerken. Nu was alleen de vraag: hoe ga ik mijn eindopdracht vormgeven? Omdat ik het zelf zo leuk vond om te experimenten, leek het mij leuk als mensen ook op mijn site zelf konden experimenteren. Daarom heb ik ervoor gekozen om gebruik te maken van sliders. Op deze manier kon ik verschillende experimenten van mij terug laten komen in mijn eindproduct en kon ik mensen de gelegenheid geven om zelf te experimenteren. 

Wat ging smoothly en wat was meer een uitdaging? 

De overstap van allemaal verschillende codepens (zie Notion, link is bovenaan te vinden) naar een geheel in visual code ging vrij soepel. Het hielp dat de HTML in de codepens overal min of meer overeen kwam (op 2 na), waardoor ze elkaar niet enorm tegenspraken of ik heel veel moest aanpassen. 1 codepen verschilde wel ten opzichte van de andere, namelijk waarin ik een 3D versie had gemaakt. Nou daar ben ik ook gelijk tegenaan gelopen, want hierdoor werkte hij niet correct in mijn visual code bestand. Toen ik de HTML structuur aangepast had en het vervolgens weer uploadde op visual code, bleek hij echter nog steeds niet helemaal te werken. Het was best wel even zoeken wat het probleem was...

Welk experiment is "gefaald"? 

Ik had met Sanne een experiment uitgewerkt waarin ik de inner divs een precies tegenovergestelde rotatie gaf, waardoor het zou lijken alsof hij meer 3D zou zijn. Eigenlijk gingen we het een beetje faken. Maar dit had ik naar mijn mening niet nodig om een interessant figuur te krijgen, ook niet in 3D. Dus dit experiment heb ik achterwegen gelaten. 
Ook wilde ik eerst met clip-path mijn tekst maken en dan binnen dat clip-path met divs eenzelfde soort patroon maken. Maar dit zag er visueel echt totaal niet uit zoals ik voor ogen had. Ik wist toen ook nog niet van het bestaan van variabele fonts :').

Wat wil ik verder nog ontdekken?

Ik zou graag nog meer willen kijken naar een 3D versie, waarbij het meer een organische vorm wordt dan dat hij nu is. Ik had eerst een idee om echt een soort 3D organische "blob" bijvoorbeeld binnen strakke lijnen te plaatsen, wat visueel iets heel interessants opleverde. Maar ik heb ervoor gekozen om me meer te focussen op de gebruikerservaring (ook een leerdoel van mij) en gebruikers zelf te laten experimenteren. Toch lijkt het me heel leuk om dit nog een keertje uit te werken.
